Application Insights from D365 F&SCM

The Microsoft reference architecture for exposing logs to Application Insights is shown below. For an overview and disambiguation of the new services that make up Azure Monitor, including the difference between Application Insights and Log Analytics, check this documentation.



For non-productive environments, you can use the Application Tracker - Emit performance telemetry from your development and test environment to Application Insights.

For Production, a custom module will need to be developed and the X++ code will need to be instrumented to call the logging methods. Sample code has been posted by Adria Ariste and Lane Swenka.

Great candidates for this additional logging are:

  • interacting with SharePoint
  • making calls to web APIs

That's because exposing errors or details about such calls are useful for technical people (Support, IT, Developers) and may not be relevant for business users.
